FUNCIONALIDADES PRINCIPAIS

1. Gerenciamento de Materiais
   - Adicionar/remover diferentes tipos (Livro, Revista, DVD)
   - Buscar por ID ou termo (titulo/autor)
   - Listar disponiveis ou emprestados
   - Prazos automaticos por tipo

2. Gerenciamento de Usuarios
   - Registrar novos usuarios
   - Rastrear emprestimos ativos
   - Validar existencia

3. Sistema de Emprestimo/Devolucao
   - Emprestar materiais
   - Registrar devolucoes
   - Controlar prazos
   - Detectar atrasados

4. Busca e Filtros
   - Busca por termo (case-insensitive)
   - Filtro por disponibilidade
   - Filtro por tipo
   - Listagem completa

PRINCIPIOS POO APLICADOS

1. Encapsulamento (Privacidade de dados)
   - Atributos privados
   - Metodos publicos controlados
   - Getters/Setters apropriados

2. Heranca (Reutilizacao)
   - Material como base abstrata
   - 3 especializacoes concretas
   - Comportamento heranca comum

3. Polimorfismo (Multiplas formas)
   - Metodo abstrato getTipo()
   - Implementacoes especificas
   - Tratamento generico via Material

4. Abstracao (Ocultamento de complexidade)
   - Material como classe abstrata
   - Interfaces Emprestavel e Buscavel
   - Servico como fachada

5. SOLID Principles
   - SRP: Cada classe uma responsabilidade
   - OCP: Aberto para extensao, fechado para modificacao
   - LSP: Substituicao de Liskov respeitada
   - ISP: Interfaces segregadas
   - DIP: Dependencia em abstraoes

CONCLUSAO

O Sistema de Gerenciamento de Biblioteca e um projeto completo que demonstra 
de forma pratica e educacional os principais conceitos de Programacao Orientada 
a Objetos em Java. 

Com uma arquitetura bem pensada, o projeto e facilmente extensivel, testavel 
e mantenivel. Todos os principios SOLID foram aplicados, garantindo qualidade 
e robustez do codigo.

Este projeto serve como excelente referencia para aprender POO ou como base 
para sistemas mais complexos de gerenciamento.
